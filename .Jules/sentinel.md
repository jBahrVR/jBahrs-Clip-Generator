## 2026-04-09 - Prevent unbounded subprocesses via timeouts\n**Vulnerability:** `subprocess.run` commands (ffmpeg, yt-dlp) lacked `timeout` arguments.\n**Learning:** Malicious inputs, hung servers, or bad network connections could cause these blocking commands to run indefinitely, leading to application freezes or a Denial of Service (DoS) state.\n**Prevention:** Always add a `timeout` parameter to `subprocess.run` calls, sizing the limit appropriately for the expected workload (e.g., 30s for metadata, 600s for heavy video encoding).
